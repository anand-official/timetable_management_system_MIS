# Timetable Management System — CLAUDE.md

## Project Overview

Build a full-stack Timetable Management System for **Modern Indian School** that automatically generates conflict-free timetables using real 2025–26 academic data.

The system must:

* Accept Excel input with teacher-subject-class assignments
* Generate conflict-free timetables for **40 sections**
* Validate teacher workloads
* Export timetables as **Excel (.xlsx) and PDF**
* Provide an interactive dashboard to view schedules by class or teacher
* Use **Gemini free API** for AI-assisted optimization
* Pre-populate database using the provided real dataset
* Provide deterministic results (avoid random scheduling instability)

* workload target

Use uploaded dataset as schema reference.

---

## 2. Timetable Generation Requirements

### Must satisfy constraints:

### Teacher Constraints

* A teacher cannot be assigned to two classes in the same period
* Teacher weekly workload must match target workload
* HOD workload should be lighter (18–22 periods)
* Max 6 periods per day per teacher
* Preferred: class teacher teaches own class

### Lab Constraints

Lab sessions must:

* always be double periods
* be consecutive
* assigned correct lab teacher
* support group rotations:

  * P/C
  * P/B
  * B/C

### Innovation Period

* exactly 1 period per week per class
* assigned to innovation incharge for grade

### Special Subjects Rotation

Subjects must rotate fairly:

* Games
* Library
* Work Experience
* Yoga
* Aerobics
* Music
* Art

### Language Slots

VI–VIII require:

* 2nd language slot
* 3rd language slot

### Daily Distribution Constraints

Avoid:

* same subject > 2 consecutive periods
* teacher with >6 periods/day
* uneven weekly subject clustering

# Known Pitfalls to Avoid

## Scheduling Issues

Avoid:

* teacher double booking
* missing lab double periods
* uneven distribution of subjects
* incomplete timetables
* empty teacher workloads
* class teacher not teaching own class

## Data Issues

Ensure:

* all teachers mapped to subjects
* no orphan subjects
* no unused teachers
* consistent abbreviations

## Algorithm Issues

Avoid:

* random scheduling
* infinite loops
* race conditions in DB insert
